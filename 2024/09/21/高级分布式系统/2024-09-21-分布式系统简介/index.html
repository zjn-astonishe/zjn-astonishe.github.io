分布式系统简介

发表于2024-09-21|更新于2024-11-09|高级分布式系统
|字数总计:4.6k|阅读时长:19分钟

分布式系统简介

定义

  • Autonomous processors communicating over a communication network
    • No common physical clock
    • No shared memory
    • Geographical seperation
    • Autonomy and heterogeneity(异构)
  • 分布式系统：物理分布，逻辑集中；个体独立，整体统一
    • 若干独立自主计算机的集合(硬件)
    • 这些计算机对于用户来说像是单个耦合系统(软件)
  • A distributed system is one in which the failure of a computer you didn’t even know existed can render your own computer unusable(不知道存在让自己计算机无法使用的故障计算机)——Leslie Lamport

特性

  • 自主性
    • 计算节点硬件或软件进程是独立的
  • 耦合性
    • 用户或者应用程序感觉系统是一个系统——节点之间需要相互协作

目标

  • 使资源可访问：让用户方便地访问资源
    • 资源访问共享
      • 基于云的存储和文件系统
      • 基于P2P的流媒体系统
      • 共享邮件系统(外包邮件系统)
      • 共享的Web支撑系统(CDN)
    • The network is the computer
  • 透明性：隐藏资源在网络上的分布(隐藏进程和资源在多台计算机分布这一事实)，分布式透明性是一个较好的属性，但是需要区别对待
    • 访问：隐藏数据表示形式的不同以及资源访问方式的不同
    • 位置：隐藏资源所在位置
    • 迁移：异常资源是否移动到另一个位置
    • 重定位：隐藏资源是否在使用过程中移动到另一个位置
    • 复制：隐藏是否对资源进行复制
    • 并发：隐藏资源是否由相互竞争的用户共享
    • 故障：隐藏资源的故障和修复
    • 持久化：隐藏数据在主存和磁盘
    • 完全的透明性是不可取的，也是难以实现的：
      • 可能掩盖通信的性能问题
      • 完全隐藏网络和节点的失效是不可能
        • 不能区分失效和性能变慢的节点
        • 不能确定系统失效之前的操作是什么
      • 完全的透明性可能牺牲性能，暴露系统分布特征
      • 保证复制节点与主节点的一致性需要时间
      • 为了容错需要立即将内存修改的内容同步到磁盘上
  • 开放性：访问接口的标准化，系统根据一系列准则提供服务，准则描述了所提供服务的语法和语义(标准化)
    • 系统应该具有良好定义的接口
    • 系统应该容易实现互操作性
    • 系统应该支持可移植性
    • 系统应该容易实现可扩展性
    • 策略与机制：机制是提供的功能，策略是如何用这些功能
      • 两者严格分离：需要设计合适的机制，导致出现很多配置参数和复杂的管理
      • 两者耦合：硬编码某些策略可以简化管理和配置参数，但迁移性灵活性不好，不同的系统要不同的策略
  • 可扩展性：系统在规模、地域、管理上的可扩展性
    • 规模可扩展性(大部分系统关心)：用户数量和进程数量增加
      • 多个服务器独立并行运行
    • 地理可扩展性：节点之间的物理位置距离扩大
    • 管理可扩展性：管理域的数量

案例

  • 局域网
  • 分布式数据库
  • 搜索集群
  • 大规模深度学习系统
  • 大规模微服务系统

架构

Motivation for Distributed System

  • Inherently distributed computation(分布式计算本就是固有的特征)
  • Resource sharing
  • Access to remote resources
  • Increased performance / cost ratio(提升性价比)
  • Reliability
    • availability
    • integrity
    • fault-tolerance
  • Scalability
  • Modularity and incremental expandability

Parallel Systems

Multiprocessor systems

  • direct access to shared memory, UMA model, like Omega, Butterfly, Clos, Shuffle-exchange networks(可直接访问内存的多处理器系统)
  • Interconnection network
    • bus(总线)
    • multi-stage switch(多阶段开关)
  • Interconnection generation function(互联功能), routing function(输出功能)
  • UMA Models(Uniform memory access)
    • the processors may locally cache data from memory
  • Omega
    • n processors, nmemory banks
    • log n stages: with n/2 switches of size 2x2 in each stage(图中小方块)
    • Interconnection function: Output i of a stage connected to input j of next stage:
    • Routing function: in any stage s at any switch:
      • to route to dest. j(要输出到内存编号的最后一位是0还是1？)
        • if $(s+1)^th$ MSB of $j = 0$ then route on upper wire
        • else $[(s+1)^th]$ MSB of $j =1$ then route on lower wire
  • Butterfly
  • Nvidia NVlink
  • Interconnection Topologies for Multiprocessors

Multicomputer parallel systems

  • no direct access to shared memory, NUMA model, like NYU Ultracomputer, CM* Connection Machine, IBM Blue gene, 天河二号(不能直接访问内存的多计算机并行系统)
    • NUMA(Non-uniform memory access)
      • the processors may locally cache data from memory

Array processors

  • colocated, tightly coupled, common system clock, like Niche market, DSP applications(处理器阵列)

Flynn’s Taxonomy(费林分类法)

  • SISD: Single Instruction Stream Single Data Stream(traditional)
  • SIMD: Single Instruction Stream Multiple Data Stream
    • scientific applications, applications on large arrays
    • vector processors矢量处理器, systolic arrays脉动阵列, Pentium/SSE, DSP chips
  • MISD: Multiple Instruction Stream Single Data Stream
    • visualization(用于可视化)
  • MIMD: Multiple Instruction Stream Multiple Data Stream
    • distributed systems, vast majority of parallel systems

Some Terminology(一些术语)

  • Coupling(联轴器)
    • Interdependency/binding among modules(模块间的相互依赖或绑定), whether hardware or software(e.g., OS, middleware)
  • Parallelism: T(1) / T(n)
    • Function of program and system
  • Concurrency(并发性) of a program
    • Measures productive CPU time vs. waiting for synchronization operations(CPU运行的时间与等待同步操作的时间的比例)
  • Granularity of a program
    • Amt. of computation vs. amt. of communication
    • Fine-grained program suited for tightly-coupled system(细粒度程序适合紧密耦合的系统)
  • Message-passing and Shared Memory
    • Emulating MP over SM(共享内存模拟消息传递)
      • Partition shared address space(部分共享地址空间)
      • Send/Receive emulated by writing/reading from special mailbox per pair of processes(向特殊的mailbox读写模拟收发)
    • Emulating SM over MP(消息传递模拟共享内存)
      • Model each shared object as a process(开一个共享进程)
      • Write to shared object emulated by sending message to owner process for the object(向该进程发送消息模拟写共享内存)
      • Read from shared object emulated by sending query to owner of shared object(向该进程接收消息模拟读共享内存)

分布式系统与其他系统的关系

Classification of Primitives(分类原语)

Synchronous and Asynchronous(同步和异步)

  • Synchronous(sned/receive, 读写都有同步的)
    • Handshake between sender and receiver(收发间需要握手)
    • Send completes when Receive completes(接收完成后发送才完成)
    • Receive completes when data copied into buffer(数据完全拷贝进入缓存后接收才完成，所以发送是最后完成的？)
  • Asynchronous(only send, 只有发送才是异步的)
    • Control returns to process when data copied out of user-specified buffer(发送不用等接收完成才返回，只用在发送的数据从用户指定的缓存区复制出去后就可返回)
  • Messgae-passing System
    • Asynchronous Executions:
      • there is no processor synchrony(处理器不同步) and there is no bound on the drift rate of processor clocks(时钟漂移率也没有限制，时钟漂移是抖动的另一种形式：当抖动频率小于10Hz的时候把这样的抖动叫做漂移。抖动即在时钟信号短时间内的波动。)
      • message delays(transmission + propagation times) are finite but unbounded(消息延迟包括传输和传播时间，有限但无界)
      • there is no upper bound on the time taken by a process to execute a step(进程执行一个步骤所花费的时间没有上限)
        • The arrows denote the message
        • The tail and head of an arrow mark the send and receive event for that message, denoted by a circle and vertical line, respectively(圆圈是发送事件，垂直线是接收事件)
        • Non-communication events, also termed as internal events, are shown by shaded circles(非通信的内部事件，有黑圈表示)
    • Synchronous Executions
      • processors are synchronized and the clock drift rate between any two processors is bounded(处理器同步，任意两个处理器的时钟漂移率是绑定的)
      • message delivery(transmission + delivery) times are such that they occur in one logical step or round
      • there is a known upper bound on the time taken by a process to execute a step(一个进程执行一个步骤所花费的时间有一个已知的上限)
  • Comparison
    • Difficul to build a truly synchronous system, can simulate this abstraction(非常困难构建一个真正的同步系统，但可以模拟同步系统)
    • Virtual synchrony(虚拟同步):
      • async exection, processes synchronize as per application requirement(异步执行，进程根据应用程序要求同步)
      • execute in rounds/steps
    • Emulations:
      • Async program on sync system: trivial(Async is special case of Sync，同步系统中的异步程序，异步是同步的一种特例)
      • Sync program on async system: tool called synchronzier(异步系统的同步程序，比较困难)
        • Assumption: failure-free system
        • System A emulated by system B(B模拟出了A，则):
          • if not solvable in B, not solvable in A(无法在B中解决的问题，A中也不能解决)
          • if solvable in A, solvable in B(能在A中解决的问题，在B中一定能解决)

Blocking and Nonblocking(阻塞和非阻塞)

  • Blocking(send/receive)
    • Control returns to invoking process after processing of primitive(whether sync or async) completes(需要在原语执行完成后才返回)
  • Nonblocking(send/receive)
    • Control returns to process immediately after invocation(调用结束后直接返回)
    • Send: even before data copied out of user buffer(甚至在数据从用户缓冲区复制出来之前)
    • Receive: even before data may have arrived from sender(甚至在数据从发送者到达之前)

Combination of Blocking/Nonblocking, Synchronous/Asynchronous, send/receive primities

Challenges

System Perspective

  • Communication mechanisms
    • Remote Procedure Call(RPC，远程过程调用)
    • Remote Object Invocation(ROI，远程对象调用)
    • Message-Oriented communication(面向消息的通信) vs. Stream-oriented communication(面向流的通信)
  • Processes:
    • Code migration(代码迁移)
    • process/thread management at clients and servers(客户端和服务端的进程/线程管理)
    • deign of software and mobile agents(设计软件和移动用户)
  • Naming:
    • Easy to use identifiers(容易使用标识符) needed to locate resources and processes transparently and scalably(透明和可扩展地运行，定位资源)
  • Synchronization(同步)
  • Data storage and access(存储数据和访问数据)
    • Schemes for data storage, search, and lookup should be fast and scalable across network(数据的存储、搜索和查找方案应该快速且可跨网络扩展)
    • Revisit file system design(需要重新审视对文件系统的设计)
  • Consistency and replication(一致性和复制)
    • Replication for fast access, scalability, avoid bottlenecks(为了快速访问和可扩展性要进行复制)
    • Reuire consistency management among replicas(副本之间要有一致性管理)
  • Fault-tolerance(容错):
    • correct and efficient operation despite link, node, process failures(即使链路、节点、流程出现故障，依然能正确有效运行)
  • Distributed systems security(安全)
    • Secure channels(信道安全)
    • access control(访问安全)
    • key management(key generation and key distribution，密钥管理：设计生成和分发)
    • authorization(认证)
    • secure group management(组管理)
  • Scalability and modularity of algorithms, data, services(可扩展性和模块化)
  • Some experimental systems
    • Globe
    • Globus
    • Grid
    • PlanetLab
  • API for communications, services: ease of use(容易使用的API)
  • Transparency: hiding implementation policies from user
    • Access: hide differences in data rep across systems, provide uniform operations to access resources
    • Location: locations of resources are transparent(要访问的资源在哪用户是不知道的)
    • Migration: relocate resources without renaming
    • Relocation: relocate resources as they are being accessed
    • Replication: hide replication from the users
    • Concurrency(并发，互斥？): mask the use of shared resources
    • Failure: reliable and fault-tolerant operation

Algorithm/Design

  • Useful execution models and frameworks: to reason with and design correct distributed programs(模型和框架，推理和设计正确的分布式模型)
    • Interleaving model(交错模型)
    • Partial order model(部分顺序模型)
    • Input/Output automata(输入/输出自动机)
    • Temporal Logic of Actions(行动的时间逻辑)
  • Dynamic distributed graph algorithms(动态分布式图算法) and routing algorithms(路由算法)
    • System topology: distributed graph, with only local neighborhood knowledge(只有本地邻域的知识构建分布式图，系统拓扑)
    • Graph algorithms: building blocks for group communication, data dissemination, object location(图算法：为组通信、数据传播、对象定位构建块)
    • Algorithms need to deal with dynamically changing graphs(解决动态变化的图)
    • Algorithm effiency: also impacts resource consumption, latency, traffic, congestion(影响资源消耗、延迟、流量和拥塞)
  • Time and global state(时间和全局的状态)
    • 3D space, 1D time
    • Physical time(clock) accuracy，物理时间的精度
    • Logical time captures inter-process dependencies and tracks relative time progression，逻辑时间捕获进程间的依赖关系并跟踪相对时间进程
    • Global state observation: inherent distributed nature of system(全球状态观测，系统固有的分布式特性)
    • Concurrency measures(并发度量): concurrency depends on program logic, execution speeds within logical threads, communication speeds(程序逻辑、逻辑线程内的执行速度、通信速度)
  • Synchronization/coordination mechanisms(同步和协作机制)
    • Physical clock synchronization: hardware drift needs correction(硬件漂移需要校正，让物理时钟同步)
    • Leader election: select a distinguished process, due to inherent symmetry(选择一个较好的进程作为领导者)
    • Mutual exclusion: coordinate access to critical resources(相互排斥，协调关键资源的获取)
    • Distributed deadlock detection and resolution: need to observe global state; avoid duplicate detection(避免重复检测), unnecessary aborts(不必要的中止)
    • Termination detection(终止检测): global state of quiescence(全局静止状态); no CPU processing(无CPU处理) and no in-transit messages(无传输中的消息)
    • Garbage collection: Reclaim objects no longer pointed to by any process(回收不再被任何进程指向的对象)
  • Group communication, multicast, and ordered message delivery(组通信、多播和有序消息传递)
    • Group: processes sharing a context, collaborating(共享上下文、协作的流程)
    • Multiple joins, leaves, fails
    • Concurrent sends: semantics of delivery order(并发发送：交付顺序)
  • Monitoring distributed events and predicates(监视分布式的事件和预测)
    • Predicate: condition on global system state(全局系统状态条件)
    • Debugging, environmental sensing, industrial process control, analyzing event streams(事件流分析)
  • Distributed program design and verification tools
  • Debugging distributed programs
  • Data replication, consistency models, and caching
    • Fast, scalable access;
    • coordinate replica updates;
    • optimize replica placement
  • World Wide Web design: caching, searching, scheduling
    • Global scale distributed system; end-users
    • Read-intensive; prefetching over caching
    • Object search and navigation are resource-intensive
    • User-perceived latency
  • Distributed shared memory abstraction(共享内存抽象)
    • Wait-free algorithm design(无等待算法设计): process completes execution, irrespective of actions of other processes, i.e., n - 1 fault-resilience
    • Mutual exclusion(相互排斥)
      • Bakery algorithm, semaphores(信号量), based on atomic hardware primitives, fast algorithms when contention-free access
    • Register constructions
      • Revisit assumptions about memory access
      • What behavior under concurrent unrestricted access to memory? 并发无限制访问内存Foundation for future architectures, decoupled with technology (semiconductor, biocomputing, quantum …)
    • Consistency models:
      • coherence versus access cost trade-off(一致性和访问成本相比)
      • Weaker models(弱模型) than strict consistency of uniprocessors(单一处理器的严格一致性)
  • Reliable and fault-tolerant distributed systems
    • Consensus algorithms(共识算法): processes reach agreement in spite of faults (under various fault models)，尽管存在故障流程仍能达成一致
    • Replication and replica management(复制和副本管理)
    • Voting and quorum(法定人数) systems
    • Distributed databases, commit: ACID properties
    • Self-stabilizing systems: ”illegal” system state changes to ”legal” state; requires built-in redundancy(内置冗余)
    • Checkpointing and recovery algorithms: roll back and restart from earlier ”saved” state(回滚并从之前的“已保存”状态重新启动)
  • Failure detectors(故障检测器):
    • Difficult to distinguish a ”slow” process/message from a failed process/ never sent message(很难区分 “缓慢进程、消息和失败进程、从未发送的消息)
    • algorithms that ”suspect” a process as having failed and converge on a determination of its up/down status
  • Load balancing(负载平衡，动态减少延迟和动态提高吞吐量): to reduce latency, increase throughput, dynamically. E.g., server farms
    • Computation migration(计算迁移): relocate processes to redistribute workload(重新分配工作负载)
    • Data migration(数据迁移): move data, based on access patterns(访问模式)
    • Distributed scheduling(分布式调度): across processors(跨处理器)
  • Real-time scheduling: difficult without global view, network delays make task harder(实时调度：没有全局视图会很困难，网络延迟会导致任务更困难)
  • Performance modeling and analysis: Network latency to access resources must be reduced(性能建模和分析：必须减少访问资源的网络延迟)
    • Metrics: theoretical measures for algorithms, practical measures for systems(度量：算法的理论度量，系统的实际度量)
    • Measurement methodologies and tools(测量方法和工具)
  • 系统设计
    • 正确的接口设计和抽象
    • 如何换分功能和可扩展性
  • 一致性
    • 如何一致共享数据
  • 容错
    • 如何保障系统在出现部分失效的情况下继续正常运行
  • 不同的部署场景
    • 集群、广域分布、传感网络
  • 实现
    • 如何最大化并行，突破性能瓶颈，平衡负载

Applications and Emerging

  • Mobile systems(移动系统)
    • Wireless communication(无线通信): unit disk model(磁盘单元); broadcast medium(MAC，广播媒体), power management(能源管理) etc.
    • CS perspective: routing(路由), location management(位置管理), channel allocation(信道分配), localization(定位) and position estimation(位置估计), mobility management(移动管理)
    • Base station model(基站模型)(cellular model，蜂窝模型)
    • Ad-hoc network model(rich in distributed garph theory problems)
  • Sensor networks(传感器网络): Processor with electro-mechanical interface(机电接口)
  • Ubiquitous or pervasive computing(普适计算)
    • Processors embedded in and seamlessly pervading environment(嵌入并无缝渗透环境的处理器)
    • Wireless sensor and actuator mechanisms(执行机制); self-organizing(自组织); network-centric(网络中心化), resource-constrained(资源受限)
    • E.g., intelligent home, smart workplace
  • Peer-to-peer computing(点对点计算):
    • No hierarchy(没有等级制度); symmetric role(对称角色); self-organizing(自组织); efficient object storage and lookup(高效的对象存储和查找); scalable(可扩展性); dynamic reconfig(动态重新配置)
  • Publish/subscribe(发布和订阅), content distribution(内容分发)
    • Filtering information to extract that of interest(过滤信息，只提取自己感兴趣的)
  • Distributed agents(分布式代理)
    • Processes that move and cooperate to perform specific tasks(处理器移动和协作执行特定的任务); coordination, controlling mobility(控制移动性), software design and interfaces
  • Distributed data mining(分布式数据挖掘)
    • Extract patterns/trends of interest(提取感兴趣的模式/趋势)
    • Data not available in a single repository(数据在单个存储库不可用)
  • Grid computing(网格计算)
    • Grid of shared computing resources; use idle CPU cycles
    • Issues: scheduling, QOS guarantees, security of machines and jobs
  • Security(安全)
    • Confidentiality(机密性), authentication(身份验证), availability(可用性) in a distributed setting
    • Manage wireless, peer-to-peer, grid environments
      • Issues: e.g., lack of trust, broadcast media, resource-constrained, lack of structure

理解的谬误

  • The network is reliable(网络不一定是可靠的)
  • Latency is ZERO(肯定有延迟)
  • Bandwidth is infinite(带宽是有限的)
  • The network is secure(网络不一定安全)
  • Topology doesn’t change(拓扑会动态改变)
  • There is only one administrator(不止一个管理员)
  • Transport costs(传输有代价)
  • The network is nomogeneous(网络有一定规则)
